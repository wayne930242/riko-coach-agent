# 建立健康基本檔案

## 步驟 1：確認是否已有檔案

用 Read 工具嘗試讀取 `data/profile.md`：
- **已存在** → 詢問使用者：「我們之前有記錄你的資料了！要更新哪個部分，還是重新填寫？」
- **不存在** → 進入步驟 2

## 步驟 2：開場說明

告訴使用者接下來要做什麼：
> 「讓我們先花幾分鐘了解你的狀況。問題有點多但都很快，準備好了嗎？」

## 步驟 3：逐一詢問（每次只問一個問題）

按照以下順序，每次只問**一個問題**，等使用者回答後再問下一個：

**【基本資料】**

1. > 「先問個基本的——你叫什麼名字？（暱稱也可以）」

2. > 「[名字]，你大概幾歲？」

3. > 「你的身高和體重大概是？（例如：170cm、65kg）不想說可以跳過」

**【健康目標】**

4. > 「你來找我主要想達成什麼？
   > A) 減脂瘦身
   > B) 增肌變壯
   > C) 提升體能耐力
   > D) 改善睡眠品質
   > E) 維持健康生活習慣
   > F) 其他（說說看）」

**【運動習慣】**

5. > 「你現在一週大概運動幾次？每次大概多久？做什麼運動？」

6. > 「你有在使用健身房嗎？
   > A) 有，固定去（請說說是哪種健身房，例如：World Gym、連鎖健身房、社區健身室）
   > B) 偶爾去
   > C) 沒有，只在家或戶外運動」

   如果有或偶爾去 → 追問：「健身房裡有哪些器材？有槓鈴架、啞鈴、纜繩機嗎？大概描述一下就好」

7. > 「家裡有沒有任何運動器材？
   > （例如：啞鈴、彈力帶、瑜珈墊、跳繩、壺鈴，或是完全沒有）」

8. > 「你比較喜歡什麼時間運動？早上、午休、下班後，還是假日？」

**【未來運動意願】**

9. > 「理想中，你希望一週運動幾天？」

10. > 「每次你能空出多少時間？（包含換衣、淋浴的話）」

11. > 「未來主要想在哪裡運動？
    > A) 繼續用現在的地點
    > B) 想加入健身房
    > C) 以居家 / 戶外為主」

**【飲食習慣】**

12. > 「飲食上有什麼特殊限制嗎？
    > A) 一般飲食，沒有限制
    > B) 素食 / 純素
    > C) 低碳 / 生酮
    > D) 有食物過敏或不耐受
    > E) 其他」

13. > 「你平常吃飯的方式是？
    > A) 大多自己煮
    > B) 大多外食（便當、小吃、餐廳）
    > C) 自煮和外食各半
    > D) 公司有供餐 / 學校餐廳」

14. > 「你住的附近食物取得方便嗎？比如超市、市場、健身餐外送這些容易找到嗎？」

**【身體組成】**

15. > 「你有沒有量過體脂率或肌肉量？比如用體脂肪機、InBody，或是健身房幫你測的？（沒有就說沒有）」

    如果有 → 追問：「大概幾 %？有沒有肌肉量的數字？」

**【身體狀況與病史】**

16. > 「有沒有目前正在影響你的舊傷？比如膝蓋、肩膀、腰椎——哪裡會痛或受限？（沒有就說沒有）」

17. > 「有沒有慢性疾病或正在服用的藥物？比如高血壓、糖尿病、甲狀腺問題等等（不方便說沒關係）」

18. > 「過去有沒有做過手術，或比較嚴重的受傷？（沒有就說沒有）」

**【裝置】**

19. > 「最後，你有在用 Garmin 或其他穿戴裝置嗎？」

## 步驟 4：確認並儲存

收集完所有資訊後，整理摘要給使用者確認：
> 「好！讓我確認一下你的資料：
> [條列資訊]
> 這樣對嗎？有需要修改的嗎？」

確認後，用 Write 工具寫入 `data/profile.md`：

```markdown
# 健康檔案

## 基本資料
- **姓名 / 暱稱**：[名字]
- **年齡**：[年齡]
- **身高 / 體重**：[身高] / [體重]（如有提供）

## 健康目標
- [目標]

## 運動習慣（現況）
- **頻率 / 時長**：[頻率、時長]
- **運動類型**：[類型]
- **偏好時間**：[早上 / 午休 / 下班後 / 假日]

## 未來運動意願
- **目標頻率**：[每週幾天]
- **每次可用時間**：[分鐘數]
- **偏好場地**：[健身房 / 居家 / 戶外]

## 飲食習慣
- **飲食限制**：[類型]
- **用餐方式**：[自煮 / 外食 / 混合 / 供餐]
- **食物取得便利性**：[描述]

## 身體組成
- **體脂率**：[數值，或「未測量」]
- **肌肉量**：[數值，或「未測量」]
- **測量方式**：[體脂肪機 / InBody / 其他，或「無」]

## 身體狀況
- **舊傷**：[部位與描述，或「無」]
- **慢性疾病 / 用藥**：[描述，或「無 / 不提供」]
- **手術或重大傷病史**：[描述，或「無」]

## 裝置
- **穿戴裝置**：[有/無，型號]

---
*建立時間：[今天日期]*
*最後更新：[今天日期]*
```

同時用 Write 工具儲存 `data/gym-profile.md`：

```markdown
# 健身環境檔案

## 健身房
- **是否使用健身房**：[有 / 偶爾 / 無]
- **健身房類型**：[名稱或描述，或「無」]
- **健身房可用器材**：[描述，或「無」]

## 居家器材
- [器材列表，或「無」]

## 訓練環境偏好
- **目前習慣**：[地點]
- **未來計劃**：[偏好場地]

---
*建立時間：[今天日期]*
*最後更新：[今天日期]*
```

## 步驟 5：委派初始分析

儲存完成後告訴使用者：
> 「資料存好了！現在讓科學家幫你做個初始評估，分析你目前的狀況和潛在風險，稍等一下 🔍」

用 Agent 工具呼叫 `sport-scientist`，傳入以下 prompt：

```
請根據以下健康檔案做初始評估，重點分析：
1. 根據目標與現況（體重、體脂、運動習慣），評估起點狀態
2. 身體狀況的潛在風險（舊傷、疾病對訓練的影響）
3. 飲食取得條件對營養規劃的限制
4. 給主對話的建議：這位使用者最需要優先關注什麼？

[貼入 data/profile.md 的完整內容]
```

## 步驟 6：整合分析結果並收尾

收到科學家的分析後，整理重點告訴使用者（不要讀論文，講人話）。

最後說：
> 「有任何變化（體重、傷況、新目標）隨時跟我說，我會幫你更新。有什麼想聊的，開口就好！」
