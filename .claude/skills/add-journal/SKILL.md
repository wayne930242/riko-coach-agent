# 新增日誌記錄

使用者輸入：`$ARGUMENTS`

## 你的任務

幫使用者將今天的記錄寫入日誌檔案 `data/journal/YYYY-MM-DD.md`（用今天的日期）。

### 步驟 1：解析輸入

從 `$ARGUMENTS` 判斷：
- **類型**：mood（心情）、workout（健身）、food（飲食）、sleep（睡眠）、note（隨筆）
- 如果沒有指定類型，從內容推斷，或詢問使用者

如果 `$ARGUMENTS` 是空的，**主動問使用者**：
> 「你今天想記錄什麼？心情、健身、飲食、睡眠，還是其他？」

### 步驟 2：檢查今日日誌是否存在

用 Glob 工具檢查 `data/journal/` 目錄，找今天日期的檔案。

**如果已存在**：讀取現有內容，在對應章節新增，不要覆蓋已有記錄。

**如果不存在**：建立新檔案，使用以下模板：

```markdown
# 健康日誌｜YYYY-MM-DD

---

## 今日總結
> （由 /plan-reviewer 自動填入，或手動更新）

---
```

### 步驟 3：寫入對應章節

根據類型，新增到對應的章節（如果章節不存在就建立）：

- `mood` → `## 💭 心情`
- `workout` → `## 💪 健身`
- `food` → `## 🥗 飲食`
- `sleep` → `## 😴 睡眠`
- `note` → `## 📝 隨筆`

格式：
```markdown
## 💭 心情

- HH:MM — [使用者的記錄內容]
```

### 步驟 4：確認並給予回饋

寫完後，用凱教練的口吻給一句溫暖的回饋，例如：
- 記錄心情 → 「感謝你的誠實記錄，有情緒才有成長 💪」
- 記錄健身 → 「太棒了！今天的汗水都值得！」
- 記錄飲食 → 「記錄是改變的第一步，繼續！」

## 注意事項
- 時間戳記用當前時間（24小時制，精確到分鐘）
- 一天只有一個日誌檔案，所有記錄都在同一個檔案裡
- 不要刪除使用者之前的記錄
